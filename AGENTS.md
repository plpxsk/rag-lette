Goal:
Chat with your documents with your own RAG

Build your own document chat system (RAG)
Chat with your PDFs and other other documents
Get reliable answers with ample quotes and sources

Follow best practices not just in implementation, but also in development
Built with a modular design and flexible storage and deployment options

Simple things should be simple. Complex things should be possible.

# Comments in code

- NEVER write comments that say "what" the code does.
- Use comments sparingly
- If using comments, explain "WHY" or mention complex interactions with other parts of code base
- If using comments, make them succinct


# Python Environment

- Always use `.venv/bin/python` to run Python scripts
- Always use `.venv/bin/pip` to install packages
- The virtual environment is located at `./.venv`
